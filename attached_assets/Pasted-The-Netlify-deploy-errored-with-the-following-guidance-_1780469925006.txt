The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: [line 50](#L50) shows the bundler error "Could not resolve 'pg'"; [line 51](#L51) points to the require in netlify/functions/api.js; and [line 60](#L60) explains the root cause: the function is using "pg" but that dependency was not installed. The bundler (esbuild) fails because it cannot find the "pg" package in node_modules ([line 70](#L70)).

Cause
- Your Netlify Function requires the "pg" package, but "pg" is not installed where Netlify expects it. Netlify does not automatically install dependencies declared inside a function folder unless configured, so esbuild cannot resolve the import.

Solution
Choose one of these fixes (recommended: Option A).

Option A — Install "pg" at the project root (recommended)
1. Verify you have a root package.json committed to the repo.
2. From your project root run:
```bash
npm install pg --save
# or
yarn add pg
```
3. Commit package.json and the lockfile (package-lock.json or yarn.lock) and push.
4. Re-deploy on Netlify.

Why: Installing at the top-level ensures Netlify will install the dependency and esbuild can resolve "pg" when bundling functions.

Option B — If you intentionally keep a package.json inside netlify/functions
1. Verify netlify/functions/package.json exists and lists "pg" in dependencies.
2. Either:
   - Add the plugin so Netlify will install dependencies inside the functions folder. In netlify.toml:
```toml
[[plugins]]
package = "@netlify/plugin-functions-install-core"
```
   - Or run npm install in that folder as part of your build command (e.g., prepend: cd netlify/functions && npm install && cd -).

3. Commit any changes and re-deploy.

Note: Before using this option, make sure netlify/functions/package.json is committed and contains "pg".

Option C — Mark "pg" as external (only if you understand runtime implications)
1. Verify "pg" is present in package.json and node_modules will be available at runtime (commit package.json and lockfile).
2. Tell Netlify's bundler not to bundle "pg" by adding to netlify.toml:
```toml
[functions]
external_node_modules = ["pg"]
```
3. Commit and re-deploy.

Important: Only mark "pg" as external if you ensure the module will be available to the function at runtime (i.e., it is installed in the deployed environment). If not, esbuild will avoid bundling but the function will still fail at runtime.
